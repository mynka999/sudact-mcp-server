# sudact-mcp

MCP-сервер для поиска судебной практики РФ через общедоступный сайт
[sudact.ru](https://sudact.ru) (СудАкт). Официального API у СудАкта нет —
сервер работает как браузер (User-Agent + сессионная кука + опрос
асинхронного поиска сайта) и разбирает HTML выдачи.

## Инструменты

| Инструмент | Назначение |
|---|---|
| `search_court_practice` | Поиск решений по тексту / статье закона / номеру дела. `scope`: `all` (ВС + арбитраж + СОЮ + мировые) или один раздел: `vsrf`, `arbitral`, `regular`, `magistrate`, `law`. Параметры дат: `date_from`, `date_to` (ДД.ММ.ГГГГ). |
| `get_court_decision` | Полный текст решения по URL или ID документа. |

## Установка

```powershell
py -m pip install --user -r requirements.txt
```

Проверка, что всё работает (делает реальные запросы к sudact.ru):

```powershell
py -X utf8 selftest.py
```

## Подключение к Claude Code

Добавить в `~/.claude.json` в блок `"mcpServers"`:

```json
"sudact": {
  "command": "C:\\Users\\USER\\AppData\\Local\\Programs\\Python\\Python312\\python.exe",
  "args": ["C:\\Users\\USER\\Documents\\Veritas\\Code\\sudact-mcp\\server.py"],
  "env": { "PYTHONUTF8": "1" }
}
```

После правки — перезапустить Claude Code, чтобы сервер подхватился.

## Потолок бесплатного поиска

Сервер закрывает сценарий «найти практику — почитать тексты — обобщить».
Но это бесплатный публичный источник, и у него есть пределы. Полезно
понимать, где free заканчивается.

### Ограничения по данным (это сам СудАкт)

- **Не всё опубликовано.** СудАкт — агрегатор того, что суды выложили в
  открытый доступ. Часть актов не публикуется или выходит с задержкой;
  полнота не гарантируется.
- **Обезличка.** Персональные данные вырезаны («ПАО Совкомбанк к ФИО1 ФИО5»),
  поэтому искать практику по конкретному физлицу почти невозможно.
- **Арбитраж отстаёт от первоисточника** (kad.arbitr.ru) по свежести и
  полноте текстов.

### Ограничения по качеству поиска (главное)

- **Поиск буквальный, без смысла и синонимов.** Ищет совпадение слов, а не
  идею. Пример: `снижение неустойки` → 100 000+ решений, а
  `снижение неустойки 333 ГК` → 0. Длинные фразы ломают выдачу — нужны
  короткие осмысленные запросы.
- **Нет осмысленного ранжирования.** Сортировка по дате / простой
  релевантности; решение ВС и проходное решение райсуда лежат вперемешку.
- **Бедные фильтры.** Нельзя отсечь по инстанции, исходу, сумме, судье —
  только текст, раздел и даты.
- **Нет готовых правовых позиций.** Возвращаются сырые решения; выжимку
  делает уже LLM поверх текстов, а не сам сервис.

### Чего нет в принципе

- Статистики и трендов («в N% дел суд снижал неустойку»), портретов судей.
- Графа ссылок на позиции вышестоящих судов, «судьбы дела» по инстанциям,
  привязки к актуальной редакции нормы.

### Технические пределы обёртки

- 10 результатов на страницу; режим `scope=all` = 4 последовательных запроса.
- Парсер хрупкий: если СудАкт сменит вёрстку, потребуется правка `server.py`.
- Официального лимита нет, но при агрессивной частоте запросов IP могут
  притормозить. Сервер ходит последовательно, без параллельного долбления.

### Когда упрётесь в потолок → платные сервисы

Caselook / Casebook / cyberjustice имеет смысл подключать (тем же
MCP-интерфейсом), когда понадобится:

- поиск **по смыслу / правовой позиции**, а не по точным словам;
- **аналитика и прогноз** (частота довода, типичные суммы взыскания);
- **фильтры** по инстанции/исходу/судье и анализ контрагента по ИНН;
- гарантия полноты и быстрые обновления.

Грубо: **free = быстрый поиск и чтение текстов с ИИ-выжимкой; платное =
умный поиск + аналитика + полнота.**
