# sudact-mcp

MCP-сервер для поиска судебной практики РФ через общедоступный сайт
[sudact.ru](https://sudact.ru) (СудАкт). Официального API у СудАкта нет —
сервер работает как браузер (User-Agent + сессионная кука + опрос
асинхронного поиска сайта) и разбирает HTML выдачи.

## Инструменты

| Инструмент | Назначение |
|---|---|
| `search_court_practice` | Поиск решений по тексту / статье закона / номеру дела. `scope`: `all` (ВС + арбитраж + СОЮ + мировые) или один раздел: `vsrf`, `arbitral`, `regular`, `magistrate`, `law`. Параметры дат: `date_from`, `date_to` (ДД.ММ.ГГГГ). |
| `get_court_decision` | Полный текст решения по URL или ID документа. |

## Установка

```powershell
py -m pip install --user -r requirements.txt
```

Проверка, что всё работает (делает реальные запросы к sudact.ru):

```powershell
py -X utf8 selftest.py
```

## Подключение к Claude Code

Добавить в `~/.claude.json` в блок `"mcpServers"`:

```json
"sudact": {
  "command": "C:\\Users\\USER\\AppData\\Local\\Programs\\Python\\Python312\\python.exe",
  "args": ["C:\\Users\\USER\\Documents\\Veritas\\Code\\sudact-mcp\\server.py"],
  "env": { "PYTHONUTF8": "1" }
}
```

После правки — перезапустить Claude Code, чтобы сервер подхватился.

## Замечания

- Полнотекстовый поиск СудАкта строгий: лучше короткие осмысленные запросы
  («снижение неустойки»), а не длинные фразы со статьями вперемешку.
- Это бесплатный публичный источник без гарантий стабильности разметки.
  Если СудАкт поменяет вёрстку — потребуется поправить парсер в `server.py`.
- Уважайте нагрузку: сервер ходит последовательно, без параллельного
  долбления сайта.
